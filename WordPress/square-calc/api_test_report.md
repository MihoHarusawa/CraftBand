# square-calc ジェネレーティブAPI & テスト報告書

このドキュメントは、WordPress上で動作する `square-calc`（縦横編みプラかごサイズ計算ツール）に実装された、URL引数制御用「ジェネレーティブAPI機能」の仕様概要およびテスト検証結果を記録したものです。

---

## 📖 1. ジェネレーティブAPI 機能概要

本機能は、AIアシスタントや外部の統合ツールからURLパラメータを通じて初期ロード値を設定し、自動計算・自動展開図描画をトリガーする仕組みです。

### 1.1 WordPress衝突回避設計（Reserved Query Variables 対策）
WordPressコアが内部で予約しているクエリ変数（`w` = Week、`d` = Day、`h` = Hour など）との競合による「404 Not Found」を防ぐため、すべてのパラメータに `cb_` プレフィックスを強制適用しています。

### 1.2 パラメータ定義一覧
| パラメータ名 | 説明 | 指定時の動作・初期値 |
| :--- | :--- | :--- |
| `cb_model` | モデル識別子 | `square` (必須)。不一致時は処理をスキップ。 |
| `cb_w` | 横幅のターゲット寸法 (mm) | コマ数適合に利用。不足時は他寸法から自動補完。 |
| `cb_d` | 奥行のターゲット寸法 (mm) | コマ数適合に利用。不足時は他寸法から自動補完。 |
| `cb_h` | 高さのターゲット寸法 (mm) | コマ数適合に利用。不足時は他寸法から自動補完。 |
| `cb_mode` | 適合計算モード | `near` (最寄り・デフォルト), `in` (内接), `out` (外接) |
| `cb_band` | バンド基本幅 (mm) | 入力フォームの「バンド幅」に自動反映。 (初期値: `15`) |
| `cb_gap` | バンド間のすき間 (mm) | 入力フォームの「すき間」に自動反映。 (初期値: `0.5`) |
| `cb_add` | 縁の折り返し加算 (コマ) | 入力フォームの「折り返しコマ」に自動反映。 (初期値: `5`) |
| `cb_rim` | 縁に重ねる本数 (本) | 入力フォームの「重ねる本数」に自動反映。 (初期値: `2`) |
| `cb_lap` | 側面の編みひも加算 (コマ) | 入力フォームの「編みひも加算コマ」に自動反映。 (初期値: `7`) |

### 1.3 アスペクト比自動補完ロジック
三次元ターゲット寸法のいずれかが欠けている場合、クラフトバッグの標準的な美しいアスペクト比に基づいて自動補完します。
* **補完基準比率**:
  * 幅と高さ ($W:H$): $\sqrt{2} : 1$ (A4縦横規格比率)
  * 幅と奥行 ($W:D$): $3 : 1$ (標準的なトートバッグ比率)
* **補完計算の流れ**:
  * **Wのみ指定**: $D = W / 3$, $H = W / \sqrt{2}$
  * **Hのみ指定**: $W = H \times \sqrt{2}$, $D = W / 3$
  * **Dのみ指定**: $W = D \times 3$, $H = W / \sqrt{2}$
  * **2箇所指定**: 残りの1箇所を上記比率に基づいて算出

### 1.4 Single-Shot JSON DOM書き出し仕様
初回ロード時の計算完了タイミングで、一度のみ `<body>` の末尾に以下の不可視 JSON データを埋め込みます。AIはこれを読み取ることで正確な適合サイズやカットリストをミリメートル単位の純粋数値として取得できます（以降のユーザー操作による再計算では更新されないSingle-Shot仕様）。
* **DOM ID**: `craftband-simulation-data`
* **JSON構造**:
  ```json
  {
    "model": "square",
    "koma": { "w": 17, "d": 5, "h": 12 },
    "size": { "w": 263.5, "d": 77.5, "h": 186.0 },
    "total_length": 14860.0,
    "cut_list": [
      { "len": 605.0, "num": 5, "name": "横ひも" },
      { "len": 791.0, "num": 17, "name": "縦ひも" },
      { "len": 751.0, "num": 14, "name": "編みひも" }
    ]
  }
  ```

## 🧪 2. テストケース & 検証結果報告

### 2.1 パラメータ指定が無い場合の初期値（デフォルト挙動）
* **検証URL**: `https://labo.com/CraftBand/temp-page/`
* **期待される動作**:
  * URLパラメータが無い場合は、初期サイズとして「B5横プリセット (幅 257mm, 高さ 182mm)」が適用され、奇数本数制限に従って自動計算されます。
  * 合計バンド幅 $15.5\text{ mm}$ (バンド15mm + すき間0.5mm) を基準とした適合計算：
    * $W = \text{toOdd}(257 / 15.5) = \mathbf{17}\text{ 本}$
    * $D = \text{toOdd}((257 / 3) / 15.5) = \text{toOdd}(5.52) = \mathbf{5}\text{ 本}$
    * $H = \text{round}(182 / 15.5) = \mathbf{12}\text{ 段}$
* **検証結果**: 💚 **SUCCESS** (画面上の表示が W=17本, D=5本, H=12段 で綺麗にロード完了)

---

### 2.2 比率補完テスト：Wのみ指定
* **検証URL**: `?cb_model=square&cb_w=250`
* **計算プロセス**:
  * $W = 250\text{ mm}$
  * $D = 250 / 3 = 83.33\text{ mm}$ に自動補完。
  * $H = 250 / \sqrt{2} = 176.78\text{ mm}$ に自動補完。
  * 合計幅 $15.5\text{ mm}$・`mode=near` (最寄り適合) で適合計算：
    * **横本数**: $250 / 15.5 = 16.13$ ➡ 最寄りの奇数 = **`17` 本**
    * **縦本数**: $83.33 / 15.5 = 5.38$ ➡ 最寄りの奇数 = **`5` 本**
    * **高さ段数**: $176.78 / 15.5 = 11.41$ ➡ 最寄りの整数 = **`11` 段**
* **検証結果**: 💚 **SUCCESS** (W=17本, D=5本, H=11段 がフォームに自動同期ロード)

---

### 2.3 比率補完テスト：Hのみ指定
* **検証URL**: `?cb_model=square&cb_h=200`
* **計算プロセス**:
  * $H = 200\text{ mm}$
  * $W = 200 \times \sqrt{2} = 282.84\text{ mm}$ に自動逆算補完。
  * $D = 282.84 / 3 = 94.28\text{ mm}$ に自動補完。
  * 合計幅 $15.5\text{ mm}$・`mode=near` で適合計算：
    * **横本数**: $282.84 / 15.5 = 18.25$ ➡ 最寄りの奇数 = **`19` 本**
    * **縦本数**: $94.28 / 15.5 = 6.08$ ➡ 最寄りの奇数 = **`7` 本**
    * **高さ段数**: $200 / 15.5 = 12.90$ ➡ 最寄りの整数 = **`13` 段**
* **検証結果**: 💚 **SUCCESS** (W=19本, D=7本, H=13段 が自動入力され完璧に描画完了)

---

### 2.4 フルパラメータ指定 ＋ 外接適合（mode=out）
* **検証URL**: `?cb_model=square&cb_w=250&cb_d=100&cb_h=200&cb_mode=out&cb_band=15&cb_gap=0&cb_add=3&cb_rim=2&cb_lap=3.5`
* **計算プロセス**:
  * バンド合計幅 = $15.0\text{ mm}$
  * 加算余裕値: 折り返し = `3` コマ, 側面板加算 = `3.5` コマ, 縁重ね = `2` 本
  * `mode=out` (目標サイズ以上の最小コマ数) で適合計算：
    * **横本数**: $250 / 15.0 = 16.67$ ➡ $16.67$ 以上の最小奇数 = **`17` 本** ($17 \times 15.0 = 255\text{mm} \ge 250$)
    * **縦本数**: $100 / 15.0 = 6.67$ ➡ $6.67$ 以上の最小奇数 = **`7` 本** ($7 \times 15.0 = 105\text{mm} \ge 100$)
    * **高さ段数**: $200 / 15.0 = 13.33$ ➡ $13.33$ 以上の最小整数 = **`14` 段** ($14 \times 15.0 = 210\text{mm} \ge 200$)
* **検証結果**: 💚 **SUCCESS**
  * バンド幅: `15`, すき間: `0`, 横 `17`本, 縦 `7`本, 高さ `14`段, 折り返し `3`コマ, 加算 `3.5`コマ, 縁 `2`本 が完全にフォームにセットされました。

### 2.5 異常値・境界条件（セキュリティ・耐久性テスト）
* **検証URL**: `?cb_model=square&cb_w=-100&cb_band=invalid_data`
* **期待される動作**:
  * バンド幅に指定された異常値文字列 `'invalid_data'` は安全に無視され、初期値の `15` mm が維持されます。
  * `cb_w` の負の値 `-100` も、適合アルゴリズム内の安全ガードを通り、負数エラーにならず最小コマ数 `1` または適切なフォールバック処理に遷移します。
* **検証結果**: 💚 **SUCCESS** (スクリプトクラッシュを一切起こさず、不正パラメータのみを綺麗に破棄して動作することを確認)

## 🛠️ 3. 実装上の最重要ノウハウ（UI同期対策）

URLパラメータから直接 JavaScript を用いて入力フォームの値を書き換えた場合：
```javascript
// 値は書き換わるが、DOMの 'input' イベントは自動発火しない
horizontal.value = String(bestW);
```
これだけでは、ツール側に実装されているステッパー処理（`input.addEventListener('input', ...)` 内で `prevValue` をクロージャ保持して偶数・奇数の変動を制御するロジック）が**同期されず、その後にユーザーがキーボードの矢印キーなどで操作した瞬間に、値が突然飛んだり不整合が発生する重大なバグ**に繋がります。

このため、代入した直後に必ず**明示的なイベント発火**を行うコードを追加し、本バグを完璧に解消しています。
```javascript
// 値を代入後、必ず 'input' イベントを明示的にトリガーする (極めて重要)
horizontal.value = String(bestW);
horizontal.dispatchEvent(new Event('input'));
```
本番環境や別ツールへの流用時には、この明示的なイベント発火（`.dispatchEvent(new Event('input'))`）のルールを厳守してください。

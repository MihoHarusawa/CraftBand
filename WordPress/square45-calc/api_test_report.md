# square45-calc ジェネレーティブAPI & テスト報告書

このドキュメントは、WordPress上で動作する `square45-calc`（北欧編み斜め立ち上げ計算ツール）に実装された、URL引数制御用「ジェネレーティブAPI機能」の仕様概要およびテスト検証結果を記録したものです。

---

## 📖 1. ジェネレーティブAPI 機能概要

本機能は、AIアシスタントや外部の統合ツールからURLパラメータを通じて初期ロード値を設定し、自動計算・自動展開図描画をトリガーする仕組みです。

### 1.1 WordPress衝突回避設計（Reserved Query Variables 対策）
WordPressコアが内部で予約しているクエリ変数（`w` = Week、`d` = Day、`h` = Hour など）との競合による「404 Not Found」を防ぐため、すべてのパラメータに `cb_` プレフィックスを強制適用しています。

### 1.2 パラメータ定義一覧
| パラメータ名 | 説明 | 指定時の動作・初期値 |
| :--- | :--- | :--- |
| `cb_model` | モデル識別子 | `square45` (必須)。不一致時は処理をスキップ。 |
| `cb_w` | 横幅のターゲット寸法 (mm) | コマ数適合に利用。不足時は他寸法から自動補完。 |
| `cb_d` | 奥行のターゲット寸法 (mm) | コマ数適合に利用。不足時は他寸法から自動補完。 |
| `cb_h` | 高さのターゲット寸法 (mm) | コマ数適合に利用。不足時は他寸法から自動補完。 |
| `cb_mode` | 適合計算モード | `near` (最寄り・デフォルト), `in` (内接), `out` (外接) |
| `cb_band` | バンド基本幅 (mm) | 入力フォームの「バンド幅」に自動反映。 (初期値: `15`) |
| `cb_gap` | バンド間のすき間 (mm) | 入力フォームの「すき間」に自動反映。 (初期値: `0.5`) |
| `cb_add` | 縁の折り返し加算 (コマ) | 入力フォームの「折り返しコマ」に自動反映。 (初期値: `3`) |

### 1.3 斜め編み（Square45）特有の適合・補完ロジック
斜め編みでは、ひもが45度の角度で配置されるため、1コマの進み方が**対角線長（$\text{diagMm} = \text{unitMm} \times \sqrt{2}$）**になります。
* **適合単位の制約**:
  * 底面のコマ数 (`W` および `D`): **整数刻み** (1, 2, 3...)
  * 高さのコマ数 (`H`): **0.5コマ刻み** (1.0, 1.5, 2.0, 2.5...)
* **補完基準比率**:
  * 幅と高さ ($W:H$): $\sqrt{2} : 1$ (A4縦横規格比率)
  * 幅と奥行 ($W:D$): $3 : 1$ (標準的なトートバッグ比率)
* **補完計算の流れ**:
  * **Wのみ指定**: $D = W / 3$, $H = W / \sqrt{2}$
  * **Hのみ指定**: $W = H \times \sqrt{2}$, $D = W / 3$
  * **Dのみ指定**: $W = D \times 3$, $H = W / \sqrt{2}$

### 1.4 Single-Shot JSON DOM書き出し仕様
初回ロード時の計算完了タイミングで、一度のみ `<body>` の末尾に以下の不可視 JSON データを埋め込みます。AIはこれを読み取ることで正確な適合サイズやカットリストをミリメートル単位の純粋数値として取得できます（以降のユーザー操作による再計算では更新されないSingle-Shot仕様）。
* **DOM ID**: `craftband-simulation-data`
* **JSON構造**:
  ```json
  {
    "model": "square45",
    "koma": { "w": 12, "d": 4, "h": 8.5 },
    "size": { "w": 263.0, "d": 87.7, "h": 186.3 },
    "total_length": 17872.0,
    "cut_list": [
      { "len": 110.1, "num": 2, "name": "ひも No.1" },
      { "len": 107.2, "num": 2, "name": "ひも No.2" },
      ...
    ]
  }
  ```

## 🧪 2. テストケース & 検証結果報告

### 2.1 パラメータ指定が無い場合の初期値（デフォルト挙動）
* **検証URL**: `https://labo.com/CraftBand/temp-page/`
* **期待される動作**:
  * URLパラメータが無い場合は、初期サイズとして「B5横プリセット (幅 257mm, 高さ 182mm)」が適用され、斜め対角線長を基準に自動適合されます。
  * バンド幅 $15\text{ mm}$ + すき間 $0.5\text{ mm}$ = 1コマ進み $15.5\text{ mm}$。
  * **対角線長**: $\text{diagMm} = 15.5 \times \sqrt{2} = \mathbf{21.92}\text{ mm}$
  * **適合コマ数計算**:
    * $W = \text{round}(257 / 21.92) = \mathbf{12}\text{ コマ}$ (実寸: 263.0mm)
    * $D = \text{round}((257 / 3) / 21.92) = \text{round}(3.91) = \mathbf{4}\text{ コマ}$ (実寸: 87.7mm)
    * $H = \text{round}(182 / 21.92) = \mathbf{8.5}\text{ コマ}$ (※0.5刻み。$182/21.92 = 8.30$ ➡ 最寄りの0.5刻みは `8.5`。実寸: 186.3mm)
* **検証結果**: 💚 **SUCCESS** (画面上の表示が W=12, D=4, H=8.5 で綺麗にロード完了)

---

### 2.2 比率補完テスト：Wのみ指定
* **検証URL**: `?cb_model=square45&cb_w=250`
* **計算プロセス**:
  * $W = 250\text{ mm}$
  * $D = 250 / 3 = 83.33\text{ mm}$ に自動補完。
  * $H = 250 / \sqrt{2} = 176.78\text{ mm}$ に自動補完。
  * 対角線長 $21.92\text{ mm}$・`mode=near` で適合計算：
    * **横コマ数**: $250 / 21.92 = 11.40$ ➡ 最寄りの整数 = **`11` コマ**
    * **縦コマ数**: $83.33 / 21.92 = 3.80$ ➡ 最寄りの整数 = **`4` コマ**
    * **高さ段数**: $176.78 / 21.92 = 8.06$ ➡ 最寄りの0.5刻み = **`8.0` コマ**
* **検証結果**: 💚 **SUCCESS** (W=11, D=4, H=8.0 がフォームに自動同期ロード)

---

### 2.3 比率補完テスト：Hのみ指定
* **検証URL**: `?cb_model=square45&cb_h=200`
* **計算プロセス**:
  * $H = 200\text{ mm}$
  * $W = 200 \times \sqrt{2} = 282.84\text{ mm}$ に自動逆算補完。
  * $D = 282.84 / 3 = 94.28\text{ mm}$ に自動補完。
  * 対角線長 $21.92\text{ mm}$・`mode=near` で適合計算：
    * **横コマ数**: $282.84 / 21.92 = 12.90$ ➡ 最寄りの整数 = **`13` コマ**
    * **縦コマ数**: $94.28 / 21.92 = 4.30$ ➡ 最寄りの整数 = **`4` コマ**
    * **高さ段数**: $200 / 21.92 = 9.12$ ➡ 最寄りの0.5刻み = **`9.0` コマ**
* **検証結果**: 💚 **SUCCESS** (W=13, D=4, H=9.0 が自動入力され完璧に描画完了)

---

### 2.4 フルパラメータ指定 ＋ 外接適合（mode=out）
* **検証URL**: `?cb_model=square45&cb_w=250&cb_d=100&cb_h=200&cb_mode=out&cb_band=15&cb_gap=0&cb_add=3`
* **計算プロセス**:
  * バンド合計幅 = $15.0\text{ mm}$ ➡ 対角線長 $\text{diagMm} = 15.0 \times \sqrt{2} = \mathbf{21.21}\text{ mm}$
  * 折り返し加算余裕 = `3` コマ
  * `mode=out` (目標サイズ以上の最小コマ数) で適合計算：
    * **横コマ数**: $250 / 21.21 = 11.79$ ➡ $11.79$ 以上の最小整数 = **`12` コマ** ($12 \times 21.21 = 254.6\text{mm} \ge 250$)
    * **縦コマ数**: $100 / 21.21 = 4.71$ ➡ $4.71$ 以上の最小整数 = **`5` コマ** ($5 \times 21.21 = 106.1\text{mm} \ge 100$)
    * **高さ段数**: $200 / 21.21 = 9.43$ ➡ $9.43$ 以上の最小0.5刻み = **`9.5` コマ** ($9.5 \times 21.21 = 201.5\text{mm} \ge 200$)
* **検証結果**: 💚 **SUCCESS** (W=12, D=5, H=9.5 が完全にフォームにロードされ描画)

---

### 2.5 異常値・境界条件（セキュリティ・耐久性テスト）
* **検証URL**: `?cb_model=square45&cb_w=-100&cb_band=invalid_data`
* **期待される動作**:
  * 異常な文字 `'invalid_data'` は安全に無視され、初期値の `15` mm が維持されます。
  * `cb_w` の負の値 `-100` も、安全ガードを通り、負数エラーにならず最小コマ数 `1` にフォールバックします。
* **検証結果**: 💚 **SUCCESS** (スクリプトクラッシュを一切起こさず、不適切な値のみを綺麗に破棄して動作することを確認)

## 🛠️ 3. 実装上の最重要ノウハウ（UI同期対策）

プログラムから直接 HTML `input` 要素の `.value` プロパティを書き換えた場合、ブラウザ仕様により `input` や `change` イベントは**自動的に発火しません**。

これにより、ツール側に実装されているステッパー処理や、折り返しcm・コマ数の相互同期処理が連動せず、**その後のユーザーの画面操作時に数値や計算にズレが発生する重大なバグ**が発生するリスクがありました。

このため、プログラムからの代入が行われた直後に、必ず**明示的なイベント発火**を行うコードを追加し、完全に同期させています。
```javascript
// 値を代入後、必ず 'input' イベントを明示的にトリガーする (極めて重要)
inpW.value = String(bestW);
inpW.dispatchEvent(new Event('input'));
```
本番環境や他のツールへこのAPIを移植・実装する際には、この明示的なイベント発火（`.dispatchEvent(new Event('input'))`）の仕様を必ず適用してください。
